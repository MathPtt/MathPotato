13/// # The i32 Api
14///
15/// The abstract syntax tree in this programming language has two specialities:
16/// - the AST is not a tree, rather a HashMap for every type where the tree sturcture is
17/// constructed using the id values of the nodes
18/// - there is no generics in the AST. Everything is typed, and due to this we have the same
19/// methods with different types, and this is the reason we have the id_and_type methods to decide
20/// what type specific method should we call.
21///
22/// The second reason is behind the fact that we have a type specific Api, like this one: i32 Api.
23/// Using this Api it is possible to manage the nodes represent i32 type things.
24///
25/// The i32 type is equvalent to Rust's i32 type. The interpreter will do i32 things when sees
26/// these types.
27pub trait I32Api {}
28impl I32Api for MathPotatoAstTree {}
