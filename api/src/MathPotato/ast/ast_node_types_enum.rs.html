1/// This enum represents the possible values in the language.
2///
3/// The language manages the types in a way where all of the types have their own HashMap to be
4/// stored. The reason for this is easier type management and not dealing with how to manage
5/// commonalities (the same fields across types like, variable name for variable types) and
6/// differences like (integer or string type values where there is difference between types). In
7/// OOO language this would be done using interfaces, but Rust is a different animal.
8///
9/// Every AST node is aware of what is its parent and children in the tree. But these can be
10/// different types, as it happens when variable declaration (name) and variable value node (the
11/// value of the variable) happens. The nodes know the Uuid value of the parent and what is the
12/// type of it. The type is marked with this AstNodeType. This way the interpreter will know in
13/// which HashMap it should do a search for the child or the parent.
14#[derive(Clone, Default, PartialEq, Eq, Debug)]
15pub enum AstNodeType {
16    I32AstNode,
17    InfixOperationAstNode,
18    #[default]
19    None,
20}
